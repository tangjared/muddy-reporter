# Reflection

**What worked best.** The split between a deterministic XBRL anomaly engine and an LLM-driven narrative layer was the single best architectural decision. The LLM, on its own, is uneven at spotting numeric inconsistencies in long filings, and it tends to drift toward generic risk-factor commentary if left unconstrained. Pre-computing anomalies (AR vs Revenue, Net Income vs Operating Cash Flow, gross-margin jumps, inventory build, goodwill ratios) and *handing them to the model as ground truth* produced sharper, more specific red flags — and gave the system a baseline of value even when the LLM is unavailable. Forcing verbatim citations and auto-demoting any "fact" or "inference" that lacked one made the verification layer real instead of cosmetic. The Streamlit UI's tabbed view (Overview / Red Flags / Financials / Claims / Open Questions) plus the printable HTML report turned out to be enough for a credible demo without needing a separate frontend stack.

**Where hallucination, overreach, or weak evidence appeared.** Even with strict prompts, the LLM occasionally paraphrased rather than quoted source text — caught and demoted by the citation gate, but it shows the gate is load-bearing. On companies with sparse or boilerplate disclosures, the model wanted to escalate generic risk-factor language ("we operate in a highly competitive industry") into red flags; the dedup + boilerplate-suppression instructions reduced this but didn't eliminate it. The system also has nothing to say about *what management is hiding by omission* — a Muddy Waters-style report often hinges on what's not in the filings (private investigators, channel checks, satellite imagery), and we deliberately have none of that.

**What I would improve next.** Three concrete next steps. (1) Add an earnings-call-transcript layer so claim-vs-counterpoint pairs can compare what the CFO *said* against what the 10-K *filed* — this is where most short-seller theses really live. (2) Pull peer financials (same SIC code) and compute *relative* anomalies, not just intra-company time-series ones; many red flags only become visible against a comparable peer group. (3) Replace the per-chunk extraction loop with a smarter section-router that pulls only the high-signal sections of a filing (Risk Factors, MD&A, Notes 1/2 on accounting policies, related-party notes, subsequent events) — currently we read uniformly from the start of each document and lose budget on cover pages.
